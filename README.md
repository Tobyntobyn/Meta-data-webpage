#### Data exchange format for camera traps

This webpage is for developers who would like to contribute to some part of the Widlife Watcher (e.g. embedded coding, db, API, MLops ...). This website provides a one-stop-shop to find out how the Wildlife Watcher metadata is structured.

## Observations

Source: [`observations-table-schema.json`](https://github.com/tdwg/camtrap-dp/blob/main/observations-table-schema.json)

Table with observations derived from the media files. Associated with deployments (`deploymentID`). Observations can mark non-animal events (camera setup, human, blank) or one or more animal observations (`observationType` = `animal`) of a certain taxon, count, life stage, sex, behavior and/or individual. Observations can be made at different levels (`observationLevel`).
